# Product Manager Best Practices

## Decision Frameworks

- **RICE Scoring**: Reach x Impact x Confidence / Effort for prioritization
- **ICE Scoring**: Impact x Confidence x Ease for quick assessment
- **MoSCoW**: Must-have / Should-have / Could-have / Won't-have for scope decisions
- **Kano Model**: Classify features as basic, performance, or delight factors

## PRD & Documentation Standards

- Start every PRD with the problem statement, not the solution
- Include success metrics with measurable targets and timelines
- Define scope explicitly: what's IN and what's OUT
- Include user stories in format: "As a [role], I want [action] so that [benefit]"
- Add technical constraints and dependencies section
- Include go/no-go criteria for launch

## Stakeholder Communication

- Lead with the recommendation, then the reasoning
- Use data to support arguments; avoid opinion-only positions
- Frame trade-offs explicitly: "We can do A (faster, less robust) or B (slower, production-ready)"
- Tailor depth to audience: executives get summary, engineers get details
- Always include next steps and owners

## Prioritization Methodology

- Maintain a living priority stack, not just a backlog
- Re-evaluate priorities weekly against current goals
- Say no by default; require justification for additions
- Track opportunity cost of what you're NOT doing
- Distinguish urgent vs important (Eisenhower matrix)

## Execution Patterns

- Ship incrementally: MVP first, iterate based on data
- Define "done" before starting (acceptance criteria)
- Run pre-mortems: "If this fails, what was the reason?"
- Measure leading indicators, not just lagging metrics
- Build feedback loops: ship → measure → learn → adjust

## Communication Templates

- **Status Update**: Progress / Blockers / Next Steps / Asks
- **Decision Request**: Context / Options / Recommendation / Timeline
- **Feature Brief**: Problem / Solution / Success Metrics / Scope / Timeline
- **Retrospective**: What worked / What didn't / Actions
